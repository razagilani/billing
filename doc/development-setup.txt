NOTE ON RENDERING DEPENDENCIES:
	Check to see if zlib installed to its common location:

	$ ls -l /usr/lib/libz.so

	If that file exists, disregard this note.

	If that file doesn't exist, run 'ls -l /usr/lib/[your architecture]/libz.so', e.g.:
    
    $ ls -l /usr/lib/x86_64-linux-gnu/libz.so
    $ ls -l /usr/lib/i386-linux-gnu/libz.so

    Once you have found the file, create a symbolic link to it:
    
    $ sudo ln -s /usr/lib/[your architecture]/libz.so /usr/lib/libz.so

    This ensures that libraries expecting zlib's dynamic library to be in /usr/lib will be able to find it.



Create Directory for Reebill Data Files
---------------------------------------
$ sudo mkdir /db-dev
$ sudo chown user:group /db-dev

Where 'user' is your development user account name and 'group' is your development user's group name

Set Up Hosts File
-----------------
To make accessing your local development web apps easier, add the following to the file /etc/hosts:

127.0.0.1	reebill-dev
127.0.0.1	dashboard-dev
127.0.0.1	nexus-dev

After configuring Apache below, you can, for example, access Reebill by putting 'reebill-dev' into your browser.
